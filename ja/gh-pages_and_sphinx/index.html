GitHub PagesとSphinx¶

このドキュメントはGitHub Pagesにホストされています。GitHub Pagesは静的ファイルをホスト可能なので、Sphinxを利用することができます。Sphinxはドキュメンテーションツールで、このドキュメントにもShinxが使われています。

ここではSphinxで作られたドキュメントをGitHub Pagesで公開する手順について記述します。

このドキュメントでは英語でドキュメントを書き、それを日本語に翻訳する手順について書いてあります。別の言語を望む場合、en、ja、英語、日本語の部分を読み換えてください。

Sphinx¶

Sphinxのインストール¶

Ubuntuを使用しているのなら、下記のコマンドを実行します。別のOSのユーザは、それに相当するコマンドを実行してください。

$ sudo apt-get install gettext
$ sudo easy_install sphinx-gettext-helper sphinx

プロジェクト作成¶

Sphinxはセットアップコマンドを持っているので、これを実行してください。

$ sphinx-quickstart

実行するとプログラムがいくつか質問をするので、それに回答します。私はこの質問のみyと回答し、他はデフォルトにしています。

You have two options for placing the build directory for Sphinx output.
Either, you use a directory "_build" within the root path, or you separate
"source" and "build" directories within the root path.
> Separate source and build directories (y/N) [n]:

このような構成になります。

.
├── Makefile
├── build
├── make.bat
└── source
     ├── _static
     ├── _templates
     ├── conf.py
     └── index.rst

多言語対応¶

多言語でドキュメントを展開するために手を加えます。

Makefile内

「# Internal variables」の下に追記

ENSPHINXOPTS   = -d $(BUILDDIR)/doctrees $(PAPEROPT_$(PAPER)) -D language=en source
JASPHINXOPTS   = -d $(BUILDDIR)/doctrees $(PAPEROPT_$(PAPER)) -D language=ja source
HTMLSPHINXOPTS   = -d $(BUILDDIR)/doctrees $(PAPEROPT_$(PAPER)) source

「html:」の項を書き換え

html:
        $(SPHINXBUILD) -b html $(ENSPHINXOPTS) $(BUILDDIR)/html/en
        $(SPHINXBUILD) -b html $(JASPHINXOPTS) $(BUILDDIR)/html/ja
        @echo
        @echo "Build finished. The HTML pages are in $(BUILDDIR)/html/$(LANGUAGE)."

genrateを追加

generate:
        rm -rf $(BUILDDIR)/*
        $(SPHINXBUILD) -b gettext $(I18NSPHINXOPTS) $(BUILDDIR)/locale
        cd source; sphinx-gettext-helper -p ../$(BUILDDIR)/locale --update --build --statistics -l ja
        $(SPHINXBUILD) -b html $(ENSPHINXOPTS) $(BUILDDIR)/html/en
        $(SPHINXBUILD) -b html $(JASPHINXOPTS) $(BUILDDIR)/html/ja

Makefileのインデントはタブを使わなければならないことに注意してください。申し訳ありませんが、私はWindowsを使用しないので、make.batの書き方は分かりません。

ドキュメントを書く¶

sourceディレクトリ以下に*.rstファイルを作成し、reStructuredText形式で記述してください。reStructuredTextについては外部のWebサイトを参照してください。rstファイルには英語を記述します。そしてこのコマンドを実行します。

$ make generate

するとlocaleディレクトリが作られ、その中に翻訳用ファイル(*.mo)が生成されます。

msgid "This document generated by Sphinx."
msgstr "このドキュメントはSphinxによって生成されました。"

もう一度このコマンドを実行してください。

$ make generate

build/html/enとbuild/html/jaを確認してください。ドキュメントが生成されています。私達はこの手順を繰り返します。

  1. rstファイルを書く
  2. 「$ make generate」を実行する
  3. moファイルを書く
  4. 「$ make generate」を実行する
  5. check the HTML
  6. 1に戻る

HTMLに反映されなくなった場合、msgidやmsgstrを誤って消していないことを確認してください。また「#, fuzzy」と書かれている場合はこれを消す必要があります。これはsourceが変更されて、再翻訳する必要がある可能性を示しています。
